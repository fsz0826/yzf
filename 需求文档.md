# 翼支付权益领取登记管理系统

## 项目背景

当前通过 Excel 手工记录翼支付权益领取情况，管理 14 个手机号码、17 种权益、18 种优惠券的领取/使用状态及快递发货信息。Excel 操作繁琐、易出错，需要一个 Web 系统替代。

## 核心业务流程

```
配置权益 → 手机号每月领取 → 自动获得优惠券 → 使用优惠券
   ↓           ↓              ↓
 每月日期范围  每自然月重置    有有效期
 过期不可领   下月可继续领
```

**关键规则**：
1. 配置哪些手机号可以领取哪些权益，并设置每月可领取日期范围（如每月1-15日）
2. 每个自然月内，在可领取日期范围内可以领取
3. 领取成功后，自动获得该权益下的优惠券
4. 下个自然月重置为未领取状态，需重新操作

**提醒规则**：
1. 未领取快到期提醒：可领取日期即将结束时（如还剩1-2天）
2. 优惠券快到期提醒：优惠券有效期即将结束时

## 数据模型

### 1. 手机号管理

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| phone_number | VARCHAR(20) | 手机号 |
| status | TINYINT | 1=启用, 0=停用 |
| created_at | DATETIME | 创建时间 |

### 2. 权益管理

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(100) | 权益名称 |
| need_grab | TINYINT | 是否需要抢（0=否, 1=是） |
| cost | DECIMAL(10,2) | 成本 |
| benefit_value | VARCHAR(100) | 收益描述 |
| recommendation | VARCHAR(200) | 领取推荐 |
| calculation_method | TEXT | 收益计算方式 |
| sort_order | INT | 排序 |
| status | TINYINT | 1=启用, 0=停用 |
| created_at | DATETIME | 创建时间 |

### 3. 权益配置表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| phone_id | INT | 关联手机号 |
| benefit_id | INT | 关联权益 |
| grab_day_start | INT | 每月可领取开始日（1-28） |
| grab_day_end | INT | 每月可领取结束日（1-28） |
| created_at | DATETIME | 创建时间 |

**说明**：配置该手机号每月可以领取该权益的日期范围。例如 grab_day_start=1, grab_day_end=15 表示每月1-15号可领取。

**唯一索引**: phone_id + benefit_id

### 4. 权益领取记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| phone_id | INT | 关联手机号 |
| benefit_id | INT | 关联权益 |
| status | ENUM | 已领取/未领取 |
| grabbed_at | DATETIME | 领取时间（未领取时为空） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**说明**：每个自然月重置为未领取状态，需重新操作。

**唯一索引**: phone_id + benefit_id

### 5. 优惠券管理

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| benefit_id | INT | 关联权益（该权益领取后获得此券） |
| name | VARCHAR(100) | 券名称 |
| need_grab | TINYINT | 是否需要抢 |
| valid_days | INT | 有效天数（领取后N天内使用） |
| valid_start | DATE | 有效期开始（固定有效期） |
| valid_end | DATE | 有效期结束（固定有效期） |
| sort_order | INT | 排序 |
| status | TINYINT | 1=启用, 0=停用 |
| created_at | DATETIME | 创建时间 |

**有效期说明**：优先使用 valid_days（领取后N天），若未设置则使用 valid_start ~ valid_end 固定有效期。

### 6. 优惠券领取记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| phone_id | INT | 关联手机号 |
| coupon_id | INT | 关联优惠券 |
| status | ENUM | 已领取/未使用/已使用/已过期 |
| grabbed_at | DATETIME | 领取时间 |
| valid_start | DATE | 实际有效期开始 |
| valid_end | DATE | 实际有效期结束 |
| used_at | DATETIME | 使用时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**唯一索引**: phone_id + coupon_id

### 7. 快递管理

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| phone_id | INT | 关联手机号 |
| tracking_number | VARCHAR(50) | 快递单号 |
| recipient_name | VARCHAR(50) | 收件人 |
| recipient_phone | VARCHAR(20) | 收件电话 |
| address | TEXT | 收件地址 |
| status | ENUM | 待发货/已发货/已签收 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 功能模块

### 1. 仪表盘

- 各号码权益领取进度统计
- 本月成本/收益汇总
- **提醒功能**：
  - 未领取快到期的权益（可领取日期即将结束，如还剩1-2天）
  - 已领取权益中的优惠券快到期（优惠券有效期即将结束）
- **搜索功能**：按手机号、权益名称搜索

### 2. 矩阵登记页面（核心）

行=权益，列=手机号，单元格=领取状态
- 点击单元格可修改状态
- 展开行可查看该权益下的优惠券
- 批量操作：一键领取/取消
- 显示每月可领取日期范围
- 当前不在可领取日期范围内时，不可操作
- **搜索功能**：按手机号、权益名称搜索

### 3. 号码管理

- 增删改查手机号
- **创建号码时**：可从已配置的权益列表中勾选该手机号可以领取的权益，并设置每月可领取日期范围
- **卡片展示**：每个号码一张卡片，点击跳转到该号码的权益列表
- **卡片信息展示**：
  - 手机号
  - 状态标签（启用/停用）
  - 已配置权益数/总权益数
  - 已领取/未领取统计
  - 本月成本/收益统计
  - 最近领取时间
  - 快递状态（如有）
- **搜索功能**：按手机号搜索

### 4. 权益管理

- 增删改查权益
- 展开行查看/增删改查该权益下的优惠券
- 配置优惠券的有效期规则
- **搜索功能**：按权益名称搜索

### 5. 快递管理

- 录入快递信息
- 更新发货/签收状态
- **搜索功能**：按手机号、快递单号、收件人搜索

### 6. 数据导入

- 支持从 Excel 导入数据
- 字段映射配置

- 环境变量配置
